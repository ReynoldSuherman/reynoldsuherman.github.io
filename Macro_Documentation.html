The Problem

mengapa fitur ini belum ada/tidak ada sama sekali di macro recoder? jawabannya cukup mudah karena macro pada dasarnya akan jalan tidak peduli di program mana atau lagi bukan program apapun dia akan mengikuti langkah,pergerakan dari mouse/input dari keyboard tanpa melihat dia di program apa dan dimana dia sekarang!

memang ini tidak ada masalahnya untuk orang pada umumnya karena tugas macro recoder memang seperti itu mudah dan ikuti saja maupun aku ganti ke program yang lain dia akan tetap jalan sesuai dengan instruksi yang di berikan.

tapi untuk saya sendiri,saya mulai melihat ini adalah sebuah masalah,mengapa ini menjadi sebuah masalah?

mudah, karena ini menggangu dan membuat saya tidak bisa meng Alt+Tab untuk pindah keprogram lain sehingga membuat efisiensi menjadi menurun

The Solution

aku bisa saja menghentikan macro itu tapi tujuan macro recoder khususnya pada saat bermain game adalah: untuk bisa melakukan grinding tanpa harus standby selama 24 jam dan bisa afk tanpa harus terkena penalti atau membuat progress grinding di game menjadi menurun!

dari situ lah terpikirlah untuk membuat sebuah project software untuk pertama kalinya yaitu macro recoder ciptaan aku sendiri

Macro Recorder buatan Reynold

untuk pembuatan software ini dikarenakan dimakan waktu yang lama serta pastinya dikarenakan aku yang masih mahasiswa dengan keahlian yang sekarang pastinya akan memerluhkan waktu yang lama dikarenakan debug dan syntax coding yang pertama kali aku pakai, aku menggunakan python dengan bantuan JSON dalam background serta bantuan AI untuk bisa mempercepat debuging dan pencarian masalah serta demi meningkatkan efisiensi

ini adalah tampilan dari Macro recoder ciptaan aku sendiri,macro recoder ini di ciptakan untuk bisa mengeksekusi instruksi macro yang sudah di rekam ke dalam program yang ditargetkan untuk bisa meningkatkan efisiensi dan tanpa harus khawatir jika macro akan tetap berjalan walaupun sudah pindah program.

untuk tampilan bisa dibilang simple dan sederhana dan ui yang diberikan terasa simple dan minimalis,ini bisa dibilang prototype karena masih ada beberapa kesalahan dalam eksekusi tapi secara sederhananya bisa berjalan namun masih ada bug yang membuat macro menjadi error

Key Features

dalam macro recoder ini ada terdapat fitur disable key,mengapa aku membuat fitur disable key? bisa dibilang pada saat dijalakan macro kadang kita suka menekan tombol di keyboard seperti enter win dan lain-lain itu membuat program yang sedang berjalan terhenti karena adanya seperti contoh tombol enter untuk chat dan membuat yang harus nya melalukan grinding malah mengetik chat yang aneh dam spam tentunya ini akan menggangu macro oleh karena itu aku ciptakan fitur ini untuk bisa menimalisir terjadinya human error karena biasanya kita kadang suka tidak sengaja menekan key seperti enter sebelum tidur dikarenakan capek atau lelah.
